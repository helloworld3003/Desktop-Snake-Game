# Security Policy

## Supported Versions

Currently, the Desktop Snake Game is in active development on the `main` branch. 

*   **`main` Branch:** Only the most recent commit on the `main` branch is actively supported with security updates. We recommend frequently pulling the latest changes.
*   **Releases/Tags:** If specific version tags (e.g., `v1.0.0`) are created in the future, only the latest major release will receive security patches.
*   **Older Versions:** Older commits, branches, or previous releases are **not** supported and will not receive security updates.

| Version/Branch | Supported          |
| -------------- | ------------------ |
| `main` (1.0.2)| :white_check_mark: |
| Older commits  | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously, especially given its interactions with the OS level (like mouse control and window manipulation). 

If you discover a security vulnerability, please **do not** open a public issue. 

Instead, please report it privately. You can do this by:
1. Reaching out directly to the maintainer via email (if provided on the GitHub profile).
2. Using GitHub's built-in "Private vulnerability reporting" feature if it is enabled for this repository.

### What to Include in Your Report
Please provide a detailed description of the vulnerability, including:
- Steps to reproduce the issue.
- The potential impact of the vulnerability.
- Any suggested mitigations or fixes.

We will endeavor to acknowledge receipt of your vulnerability report as soon as possible and provide regular updates on our progress toward a fix.

Thank you for helping keep the project safe!
